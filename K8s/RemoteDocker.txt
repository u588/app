1: Prerequisites
    Run docker context to verify that your Docker client supports contexts.

 You will also need one of the following:
    a. Docker Swarm cluster
    b. Single-engine Docker node
    c. Kubernetes cluste

2: A context is a combination of several properties. These include:
    a. Name
    b. Endpoint configuration
    c. TLS info
    d. Orchestrator

--> docker context ls 
    * active context

context结构 --> docker context inspect default

3: Create a new context
    docker context create k8s-master-66 \
  --default-stack-orchestrator=kubernetes \
  --kubernetes config-file=/home/ts/.kube/config \
  --docker host=unix:///var/run/docker.sock

4: Use a different context
    docker context use k8s-master-66
